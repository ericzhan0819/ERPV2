## 重要契約

- Request 在套用 `UsernameRules::rules()` 前，必須先用 `UsernameRules::normalizeInput()` 正規化 username。
- `UpdateCurrentUserProfileRequest` 的 `username` 使用 `present`：缺鍵是 invalid request，`null` 是清空，兩者不得混用。
- Eloquent 寫入仍由 `User` username mutator 作最後正規化防線，Factory 與 Service 不得繞過。
- Application unique rule 提供一般 422；database unique index 才是競態時的最終正確性保護。
- 寫入 username 的正式 Service 路徑必須使用 `UserService::setUsername()`，讓 duplicate-key race 維持欄位型 validation error。

## 後續邊界

- 第 3 部分調整 `LoginRequest` 與 `AuthService` 時，沿用 `User::normalizeUsername()`；不得把 profile 的 unique rule 套到登入輸入。
- 第 4 部分 Rate Limiter 必須讓同一 User 的 username／Email 共用 canonical account key。
- 第 7 部分建立 self profile Controller／Service 時，直接掛用 `UpdateCurrentUserProfileRequest`；Controller／Service 必須只使用 validated data，且不得放寬既有 `present`／`missing` 契約。
