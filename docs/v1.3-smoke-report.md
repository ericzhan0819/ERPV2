## 2. 自動驗證涵蓋

- 正式收車人／賣車人欄位與 idempotency payload。
- 版本化獎金方案、1／3／5 台整月跨級與歷史不可回改。
- approved-only 毛利、pending／rejected 排除、虧損與整數餘數。
- 底薪、津貼、勞健保、手動加扣、員工及全公司 totals。
- 公司保留與公司剩餘直接保存 calculator 的整月批次 totals。
- 無 active salary profile／未啟用獎金時的非阻擋提示與公司剩餘歸屬。
- stale preview、確認鎖定、paid DB trigger、發薪 transaction 與 idempotency race。
- admin-only API／Resource 白名單及 manager／sales 原始 JSON 遮蔽。
- 前端 admin-only Sidebar／route guard、集中 API、錯誤與空狀態、action-level 重複送出保護。

## 3. 待使用者 Browser Manual Smoke

以下項目保持未勾選，完成後請同步 `PLAN_v1.3.md` 第 14、17 部分。

### 顯示與裝置

- [ ] 手機寬度可基本查看與操作。
- [ ] light／dark mode 文字、badge、表格、modal 與警告皆可讀。

### 前置資料

- [ ] 建立至少兩位員工薪資設定。
- [ ] 建立／確認 2026 標準薪資方案。
- [ ] 建立至少五台同一賣車人、同月份 sold 車輛。
- [ ] 至少一台收車人與賣車人不同，至少一台同人收車又賣車。
- [ ] 每台 approved 購車付款與 purchase_price 一致，approved 收款達 sold_price。

### 跨級與公式

- [ ] 1～2 台為 20%，第 3 台後整月 30%，第 5 台後整月五台全部 50%。
- [ ] 100,000 毛利顯示公司保留 40,000、收車獎金 12,000、50% 賣車獎金 30,000、公司剩餘 18,000。
- [ ] 公司保留＋所有獎金＋公司剩餘＝毛利。
- [ ] 同一人收車又賣車時同時取得兩項獎金。
- [ ] 無 active salary profile 或未啟用獎金時顯示非阻擋提示，且相應獎金歸公司剩餘。

### 薪資與異常

- [ ] 底薪、固定津貼、勞保、健保、手動加給、手動扣款、實發薪資與全公司總額正確。
- [ ] 缺收車人、缺賣車人、pending 收支、購車付款不一致皆不可確認。
- [ ] 虧損車不產生負獎金。

### 確認、發薪與權限

- [ ] 確認後只讀，重新整理後狀態保持。
- [ ] 發薪為每位正數實發員工建立一筆 MoneyEntry，帳戶餘額正確下降。
- [ ] 重複點擊／同 key 重試不重複建立支出，salary MoneyEntry 無法由一般收支修改／刪除。
- [ ] admin 可完整操作；manager／sales 無 Sidebar 入口、直接路由與 API 存取。
- [ ] manager／sales 的原始 JSON 不含任何薪資資料。
