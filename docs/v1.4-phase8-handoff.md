## 1. 本階段完成內容

- viewport 啟用 `viewport-fit=cover`，文件語系修正為 `zh-Hant`，並加入可由 ThemeProvider 同步的 `theme-color`。
- `html`、`body`、React root 與 App Shell 共用正式 `bg` 語意底色；App Shell 使用 `100dvh` 並保留 `100vh` fallback，避免行動瀏覽器工具列造成底部灰邊。
- Header 承接頂部與左右 safe area；Mobile Sidebar 承接頂部、底部與左側 safe area，導覽項目可在短 viewport 內獨立捲動。
- 主內容保留既有響應式水平內距後再加上左右 safe area，橫向 iPhone 的瀏海與圓角不會遮住內容；各表面底色仍延伸至螢幕邊緣。
- 主內容底部固定保留 32px，再加上 `safe-area-inset-bottom`，避免最後一個控制項貼近 Home Indicator。
- light／dark mode 切換同步瀏覽器 `theme-color`，App 背景、Header、Sidebar 與底部延伸區維持各自正式語意表面。
- 全域 Header 仍只有 Sidebar 開關、使用者資訊、主題切換與登出；沒有加入 Dashboard 快捷功能、通知或通知預留位，也沒有新增 Footer／底部導覽。
- 保留既有 Mobile Sidebar 的 overlay、focus trap、`Escape`、背景捲動鎖定、focus 歸還與 route change 關閉行為。
- 未修改 backend、API、database schema、Business Logic、Workflow、角色或權限。

使用 Firefox 152 headless + WebDriver BiDi、獨立 `/tmp` SQLite database、獨立 8001／5184／4444 服務與 seed admin；驗證後停止服務並刪除測試 database，未接觸既有開發資料庫。

## 5. Adversarial review 修正

Review 指出 `viewport-fit=cover` 已把四向 Safe Area 責任交給應用程式，但首版只處理 top／bottom。查證 finding 有效：橫向瀏海 iPhone 的 `safe-area-inset-left/right` 可能非 0，會使 Sidebar、Header 與主內容貼近或進入不可安全顯示區。

最小修正如下：

- Mobile Sidebar 增加左側 inset；深色 Sidebar 背景仍保持滿版，只讓內容內縮。
- Header 與主內容在既有 12／16px Mobile、24px Tablet／Desktop 水平內距之外，再分別加上左右 inset。
- UI 契約改為明確檢查四向 inset 與 landscape 瀏海／圓角。
- 重新執行 lint、typecheck、production build、`git diff --check` 與 Firefox 五種寬度 smoke；結果維持通過。Headless inset 為 0，因此真實 landscape iPhone 仍列入第 11 部分實機 smoke。
