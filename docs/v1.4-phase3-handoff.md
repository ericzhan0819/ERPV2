# ERPV2 v1.4 第 3 部分交接

日期：2026-07-21

範圍：`PLAN_v1.4.md` 第 3 部分「Dashboard API」

狀態：Dashboard 工作 KPI、經營 KPI、30 天趨勢、後端角色遮蔽與自動回歸完成；尚未進入第 4 部分 Dashboard 前端改版。

## 1. 本階段完成內容

- `DashboardService` 保留既有 REPEATABLE READ transaction，同一份資料快照內計算全部 KPI 與趨勢。
- 工作概況新增待整備、待上架、待交車及 admin-only 待審核收支。
- 經營概況新增在庫數、本月 approved-only 毛利及既有現金餘額、收入、支出與成交數。
- 近 30 天固定含今天與前 29 天，依 `Asia/Taipei` 回傳連續日期、成交量、approved-only 毛利及 cash 類型帳戶每日結束餘額。
- 新增 `DashboardSummaryResource`：admin／manager 取得完整財務資料；sales／未知角色的原始 JSON 不含財務 KPI 或財務趨勢；待審核收支只對 admin 回傳。
- 新增 `DashboardTest`，涵蓋 KPI 定義、approved-only、角色遮蔽、日期邊界、補零與現金餘額延續。
- 同步 `frontend/src/types/dashboard.ts` 與 `backend/API.md`。第 4 部分切換畫面前暫時保留舊 Dashboard 使用的頂層相容欄位，財務相容欄位仍由 Resource 套用相同遮蔽。

## 2. v1.4 API 契約

正式新畫面只讀取：

```text
work_overview
├─ preparation_pending_count
├─ listing_pending_count
├─ delivery_pending_count
└─ pending_money_entry_count（admin only）

business_overview
├─ inventory_count
└─ cash_balance／monthly_income／monthly_expense／monthly_gross_profit／monthly_sold_count（admin／manager）

trends
├─ sales_count（所有角色）
├─ gross_profit（admin／manager）
└─ cash_balance（admin／manager）
```

每個趨勢陣列固定 30 點，日期遞增，第一點為今天減 29 天，最後一點為今天。

## 3. 重要計算口徑

- 月份與日期邊界使用半開區間，避免月底微秒或下一月零點誤計。
- 本月毛利與每日毛利先選出成交車，再依該車所有 approved MoneyEntry 計算收入減支出；收支日期不會改變毛利歸屬日，歸屬日固定是車輛 `sold_at`。
- 每日現金餘額以所有 cash 類型帳戶期初餘額，加上該日結束前所有 approved 收支累計；無異動日延續前日餘額。
- pending／rejected 不進現金、收入、支出或毛利；bank／other 帳戶不進現金趨勢。
- v1.4 沒有 migration、schema、Workflow、角色或權限變更。
