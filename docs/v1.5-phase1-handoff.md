# ERPV2 v1.5 第 0～1 部分交接

日期：2026-07-23

## 完成範圍

- 完成 `PLAN_v1.5.md` 第 0 部分前置盤點與基準驗證。
- 完成第 1 部分 User Schema、Model、Factory、Seeder 與自動測試。
- 未提前實作第 2 部分 username 完整驗證、雙登入、強制改密碼流程或前端 UI。

## 實作內容

- 新增 `users.username` nullable string 與 `users_username_unique` unique index。
- 新增 `users.must_change_password` boolean，database default 為 `false`。
- Migration 不回填 username、不推測既有帳號狀態，並可在 MySQL 部分 DDL 後安全重跑。
- Down migration 先移除 unique index，再移除兩個欄位。
- `User` 加入 fillable、boolean cast，並以 `User::normalizeUsername()` 作為後續 Request／Service 與 Factory 共用的正規化邊界。
- Factory 預設維持既有測試語意，新增 `mustChangePassword()` 與 `withUsername()` states。
- `AdminUserSeeder` 明確維持 `must_change_password=false`，重跑時不覆蓋已設定的 username。
