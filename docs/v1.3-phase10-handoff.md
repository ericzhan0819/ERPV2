# v1.3 第 10 部分交接：Salary Period API Routes

日期：2026-07-15

## 本階段範圍

本階段完成 `PLAN_v1.3.md` 第 10 部分與 12.9 HTTP 邊界測試，只開放既有第 7～9 部分已完成之 Salary Period Service／Policy／Request／Resource 契約；未進入第 11 部分管理前端，也未執行完整 v1.3 manual smoke。

## 已完成

- 新增 `SalaryPeriodController`，提供月份列表、建立草稿、詳情、重算、手動加扣新增／刪除、確認與發薪。
- `SalaryPeriodService` 新增月份列表與單月讀取入口；列表以 `withCount`／`withSum` 取得員工數與實發合計。
- 正式掛載 8 條 `/api/salary-periods` routes；既有 `PATCH /api/vehicles/{vehicle}/commission-attribution` 明確加上 `role:admin`。
- index 固定使用 `SalaryPeriodListResource`，不對每列 draft 呼叫 `SalaryEligibilityService::inspectPeriod()`；詳情才使用完整 `SalaryPeriodResource`。
- 新增手動加扣時，只能由 URL 指定月份內的 `user_id` 找到 settlement；不存在回 `404`。
- 刪除手動加扣時，先確認 item 的 settlement 屬於 URL 月份，再以該 settlement 執行 `deleteAdjustment` Policy；跨月份 ID 組合回 `404`。
- `role` middleware 排在 route model binding 前，manager／sales 對敏感端點不論資源 ID 是否存在皆固定 `403`，避免用 `403`／`404` 枚舉月份或車輛歸屬資源。
- HTTP 回歸覆蓋 admin 完整流程、manager／sales 每條端點 403、raw JSON 白名單、輕量列表無車輛／收支掃描、跨月份刪除保護。

## API 契約摘要

```text
GET    /api/salary-periods
POST   /api/salary-periods
GET    /api/salary-periods/{salaryPeriod}
POST   /api/salary-periods/{salaryPeriod}/recalculate
POST   /api/salary-periods/{salaryPeriod}/adjustments
DELETE /api/salary-periods/{salaryPeriod}/adjustments/{item}
POST   /api/salary-periods/{salaryPeriod}/confirm
POST   /api/salary-periods/{salaryPeriod}/pay
PATCH  /api/vehicles/{vehicle}/commission-attribution
```

全部位於 `auth:sanctum` + `active` 安全邊界；薪資 routes 再位於 `role:admin` 群組並使用 Policy。Request／Response 細節見 `backend/API.md` 第 16 節。
