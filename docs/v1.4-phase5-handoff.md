# ERPV2 v1.4 第 5 部分交接

日期：2026-07-21

範圍：`PLAN_v1.4.md` 第 5 部分「Vehicle List API 與 Resource」

狀態：後台車輛列表封面縮圖最小契約、受限 eager load、角色遮蔽與查詢回歸測試完成；下一階段為第 6 部分 Vehicle Card Grid。

## 1. 本階段完成內容

- `Vehicle::coverPhoto()` 只關聯已提交、未 soft delete 且 `is_cover=true` 的單張照片。
- `VehicleService::listVehicles()` 在既有收／賣車人 eager load 外，固定以單一 query 預載封面，且只選取 `id`、`vehicle_id`、`disk`、`thumbnail_path`。
- 後台 `GET /api/vehicles` 改用獨立 `VehicleListResource`，保留 `VehicleResource` 的既有車輛欄位與角色遮蔽，再新增固定的 `cover_photo`。
- `cover_photo` 只含 `id` 與 `thumbnail_url`；沒有正式封面或照片仍在未完成 upload batch 時明確為 `null`。
- 列表不回傳 `photos`、原圖 URL、原始檔名、mime、上傳者等相簿／內部照片資料。
- 前端新增 `VehicleCoverPhoto` 與 `VehicleListItem` 型別，列表 response 與後端契約同步；本階段不提前修改 Vehicle List 畫面。

## 2. 權限與相容性

- admin／manager 既有 `purchase_price` 與銷售價格欄位契約不變。
- sales 仍可取得議價所需的開價／底價／成交價，但原始 JSON 不含 `purchase_price`；第 6 部分 Card 必須依企劃只呈現開價，不呈現底價。
- 列表沒有新增毛利、summary、money entries、cash account 或完整相簿。
- 詳情、列印、待補歸屬與公開官網 Resource 沒有改成列表 Resource，既有契約不受影響。
- 未新增 migration、schema、Workflow、狀態、角色或權限。

## 4. 測試鎖定的契約

- 有封面時只回傳縮圖最小欄位，且不回傳完整相簿。
- 無照片與未完成 upload batch 的封面皆回傳 `null`。
- 三台車同頁只執行一次 `vehicle_photos` 查詢；查詢不選取 `path` 或 `original_filename`。
- admin／manager／sales 的既有價格遮蔽不變，且列表不新增其他財務摘要。

## 5. 未執行與後續入口

- 本階段沒有 UI 行為變更，因此未做新的 browser manual smoke；第 6 部分 Card Grid 完成後再驗證圖片／空圖、三角色欄位、light／dark 與指定 RWD 寬度。
- 第 6 部分應直接使用 `VehicleListItem.cover_photo?.thumbnail_url` 呈現封面；`null` 時依 `UI.md` 顯示「尚無照片」空圖，不可使用假車圖。
- Vehicle Card 對 admin／manager 顯示底價、sales 不顯示底價；所有角色都不得在 Card 顯示收購價或毛利。後端詳細頁的既有銷售議價權限不因 Card 呈現改變。
