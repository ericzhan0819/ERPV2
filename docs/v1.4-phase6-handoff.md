# ERPV2 v1.4 第 6 部分交接

日期：2026-07-21

範圍：`PLAN_v1.4.md` 第 6 部分「Vehicle Card Grid」

狀態：車輛工作區 Card Grid、角色欄位差異、空圖／列表狀態與響應式分頁完成；下一階段為第 7 部分 Filter UI 與 Mobile Drawer 的其餘收斂工作。

## 1. 本階段完成內容

- 車輛工作區移除既有 Table 與 `overflow-x-auto`，改為完整可點擊並導向既有詳細頁的 Vehicle Card。
- Card 以 `VehicleListItem.cover_photo.thumbnail_url` 顯示 `4:3`、`object-cover` 封面；`cover_photo=null` 或圖片載入失敗時顯示車輛圖示與「尚無照片」，不使用假車照片。
- Card 固定顯示品牌、車型、年份、顏色、車牌、狀態 Badge 與開價；缺值使用 `—`，不偽造資料。
- admin／manager 額外顯示底價；sales 不渲染底價。所有角色的列表卡片都不顯示收購價、成交價或毛利。
- Grid 採 mobile-first：`<640px` 1 欄、`640～1023px` 2 欄、Desktop 3 欄、寬桌機 4 欄。
- loading 改為 Card skeleton；API error、無資料、無搜尋結果與超出頁碼皆使用 Grid 全寬狀態，不回退成 Table。
- 分頁在 Mobile 改為可觸控的雙欄按鈕，Desktop 維持水平排列；頁面標題與既有動作亦可換行，避免窄螢幕 overflow。
- 初始三個在庫狀態、sold／cancelled 手動 Filter、Dashboard query 覆蓋與 `is_preparation_completed` 分流沿用第 2／4 部分既有 URL 契約，本階段 browser smoke 再次確認。

## 2. 權限與相容性

- 底價的列表呈現以既有 admin／manager 財務角色白名單判斷；未知角色 fail-closed，不會取得額外顯示。
- sales 後端原始 JSON 的既有議價欄位契約沒有變更，本階段只依 v1.4 規格縮減列表卡片的可見內容。
- 後端 `VehicleListResource` 仍是敏感資料正式邊界；沒有因 UI 改版新增 API、Resource 欄位、完整相簿或財務摘要。
- 車輛詳細頁、照片管理、收支、整備、上架、保留、收款及成交 Workflow 均未修改。
- 未新增 dependency、migration、schema、Workflow、狀態、角色或權限。
