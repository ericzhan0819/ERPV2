# ERPV2 v1.4 第 4 部分交接

日期：2026-07-21

範圍：`PLAN_v1.4.md` 第 4 部分「Dashboard 前端」

狀態：Dashboard 四區塊、角色呈現、KPI 導流、三項趨勢圖、列表時序與超頁空狀態完成；後端與前端自動驗證及針對性 browser smoke 通過。下一階段為第 5 部分 Vehicle List API 與 Resource。

## 1. 本階段完成內容

- Dashboard 改為 Action Bar、工作概況、經營概況、趨勢分析四區塊，移除舊資金混排、車輛狀態與薪資卡。
- Action Bar 固定重用建立車輛、一般收支、建立客戶與 preparing 車輛工作區；sales 不顯示建立收車。
- 工作 KPI 使用完整可點擊 `Link`，沿用第 2 部分 URL Filter 契約導向待整備、待上架、待交車與 admin-only 待審核收支。
- 經營 KPI 的在庫數、現金、本月收入／支出／毛利／成交依角色呈現；月份卡明示完整當月口徑，現金卡明示帳面餘額。
- 收入／支出卡帶入台北當月起訖日、direction 與 approved 審核狀態；毛利／成交導向既有 sold 車輛工作區。
- 新增無額外 dependency 的響應式 SVG 趨勢圖。圖表具有標題、說明、單位、資料點、最低／最高／最新摘要及螢幕閱讀器資料清單，不只依靠顏色傳達。
- sales 只渲染三個工作 KPI、在庫數與成交量趨勢；manager 不渲染待審核 KPI；admin 顯示全部指定內容。正式財務遮蔽仍由後端 Resource 負責。
- MoneyEntry 核准／駁回完成後改由 refresh token 觸發 effect，以當下 URL Filter 重查，不再呼叫捕獲舊 Filter 的 reload closure。
- Vehicle／MoneyEntry 超出 `last_page` 時顯示「此頁沒有資料」與「回到第 1 頁」，回頁時保留既有 Filter。
- 新 Dashboard 切換完成後移除 API、Service 與 TypeScript 的舊頂層相容欄位，只保留 `work_overview`、`business_overview`、`trends`。

## 2. 重要契約與邊界

- Dashboard 前端不自行加總正式金額或趨勢；所有數值直接呈現 `GET /api/dashboard/summary` 回應。
- `business_overview` 的月份 KPI 可能包含同月未來日期資料；三項趨勢固定截至今日。畫面已以「完整當月」與「截至今日」分開標示。
- 圖表只負責展示 30 個後端資料點，不改變毛利、現金或成交公式。
- 未新增 chart package、route、API、migration、schema、Workflow、狀態、角色或權限。
- 本階段移除的舊 Dashboard 頂層欄位是第 3 部分明確標記的短期相容輸出；repo 內呼叫端與測試已全部切換至正式三區塊契約。
- `企劃書_v1.4.md` 5.3 要求月份 KPI 導流保留月份，但 `PLAN_v1.4.md` 第 2／9 部分的 Vehicle Filter 契約未包含成交月份。第 4 部分不自行新增 Vehicle 日期 Filter，因此本月毛利／成交目前只能保留 `status=sold`；是否在第 5／6 部分擴充成交月份 Filter，須由使用者裁決後修訂企劃／PLAN。

## 2.1 Adversarial review 修正

- 本月收入／支出連結補上 `approval=approved`，使落地列表與 KPI 的 approved-only 正式口徑一致。
- 常數趨勢序列改畫在繪圖區垂直中央；全 0 成交量或無異動現金不再貼齊圖表頂端。
- 空趨勢序列以 0 作為摘要基準並顯示「目前沒有趨勢資料」，不再可能產生 Infinity。
- admin-only 待審核 KPI 改用既有 `canApproveMoneyEntries()` 白名單 helper，不在頁面散落角色字串判斷。
- 趨勢 Grid 依實際圖表數使用 1／2／3 欄，避免 sales 單圖與 admin／manager 第三圖留下半版空白。
