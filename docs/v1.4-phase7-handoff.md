# ERPV2 v1.4 第 7 部分交接

日期：2026-07-22

範圍：`PLAN_v1.4.md` 第 7 部分「Filter UI 與 Mobile Drawer」

狀態：Vehicle／MoneyEntry 的 Desktop／Tablet Filter、Mobile Drawer、作用中條件與搜尋 debounce 已完成；下一階段為第 8 部分 App Layout、Header 與 Safe Area。

## 1. 本階段完成內容

- 新增共用 `MobileFilterDrawer`，兩個列表共用 overlay、focus trap、`Escape` 關閉、背景捲動鎖定、Desktop breakpoint 自動關閉、focus 歸還與 safe-area footer。
- Vehicle／MoneyEntry 在 `sm` 以上顯示 inline Filter，Mobile 只顯示 Drawer 入口；Drawer 開啟時從正式 URL Filter 複製 draft，只有「套用篩選」才寫回 URL。
- MoneyEntry Filter 補齊永遠可見的搜尋、方向、分類、資金帳戶、關聯車輛、起訖日期與審核狀態 label；既有 URL key、API 參數與資料可見範圍不變。
- 新增共用作用中條件 chip。Vehicle 顯示搜尋、狀態、整備狀態與成交月份；MoneyEntry 顯示全部八類正式 Filter，且每項可單獨清除。
- Vehicle 最後一個已勾選狀態會 disabled，並顯示「至少保留一個車輛狀態」，不再無提示跳回三個預設狀態；清除全部仍正式回到 preparing／listed／reserved。
- Vehicle／MoneyEntry Desktop 搜尋改為 275ms debounce，URL 更新使用 replace 且重設 page；共用輸入在 composition 期間不啟動 debounce，composition end 後才送出正式字串。
- 列表仍由既有分頁 API 取得；未新增 endpoint、dependency、後端欄位、migration、Workflow、角色或權限。

## 3. Firefox 針對性 smoke

使用 Firefox 152 headless + WebDriver、獨立 `/tmp` SQLite database、獨立 8001／5184／4444 服務與 seed admin；驗證後已停止服務並刪除測試 database，未接觸既有開發資料庫。

- Vehicle／MoneyEntry 搜尋不會立即改 URL，275ms 後才同步 URL 與查詢。
- Vehicle URL 可還原搜尋與狀態，狀態變更後 browser back 可還原前一組 Filter。
- Vehicle Drawer 可清除 draft、選為 sold-only、套用至 URL 並顯示「狀態：已售出」chip；最後一個狀態為 disabled。
- MoneyEntry `/money-entries?approval=pending` 顯示作用中條件，Drawer 新增搜尋後可同時保留 approval 並寫回 URL。
- Drawer 開啟後 focus 在關閉按鈕、body scroll 被鎖定；`Escape` 關閉後 focus 回到觸發按鈕。
- 模擬 composition start／input 後等待超過 debounce 時間，URL 不變；composition end 後才同步完整中文字。
- Vehicle／MoneyEntry 在 320、375、390、768、1440px 均無整頁水平 overflow。
