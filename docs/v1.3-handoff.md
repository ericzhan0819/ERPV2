## 部署狀態與邊界

- 本機開發 MySQL 已增量套用 migration `000011`～`000013`，沒有執行 `migrate:fresh`。
- `000013` 實質 forward-only：若升級前已有 confirmed／paid 月份會在 DDL 前 fail-fast；不可用 0 或 items 推測回填。
- 升級前遺留 draft 的公司 totals 為 null，第一次 confirm 會要求先重算，符合安全預期。
- v1.3 不包含打卡、排班、請假、官方勞健保引擎、稅務、薪轉檔、員工薪資單或正式會計。

## 使用者下一步

只需依 `docs/v1.3-smoke-report.md` 第 3 節完成 browser manual smoke。重點順序：

1. 依真實資料再覆核 light／dark mode 與手機寬度。
2. 建立兩位員工與五台同月成交車，驗證 20%／30%／50% 整月追溯。
3. 驗證公司與個人金額、非阻擋獎金設定提示、手動加扣。
4. 驗證四種阻擋異常、確認只讀、發薪與重試。
5. 以 manager／sales 驗證 UI、直接路由與原始 API JSON。
