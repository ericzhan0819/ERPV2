# v1.3 第 4 部分交接：車輛獎金歸屬

日期：2026-07-13

## 完成範圍

`PLAN_v1.3.md` 第 4 部分已完成：

- admin／manager 建車必須指定 active user 作為 `purchase_agent_id`。
- sales 執行保留流程時由後端固定歸屬本人；admin／manager 代登必須指定 `sales_agent_id`。
- 建車與 reservation 冪等比較均包含 agent id，同 key 換人回傳 `422`。
- 成交結案前必須已有正式賣車人。
- 新增獎金人員安全選項 API、admin-only 歷史待補清單與歸屬 PATCH。
- confirmed／paid salary period 已引用車輛不可更改歸屬；成功修改寫入 Vehicle Audit Log。
- 前端新增建車收車人、銷售賣車人與 admin 待補歸屬頁面。

車輛歸屬代表實際經手人的事實資料，只要求 `users.is_active=true`；是否有 Salary Profile、是否 `commission_enabled` 由薪資結算階段判斷，不得阻斷建車或保留流程。這也讓 fresh seed、v1.2 升級環境及不抽成的 sales 能維持核心營運。歷史資料仍不使用 `created_by`／`updated_by` 推定。獎金人員選項只開放 admin／manager，sales 不可讀取。

## API

- `GET /api/vehicles/commission-agent-options`
- `GET /api/vehicles/commission-attribution-pending`（admin-only）
- `PATCH /api/vehicles/{vehicle}/commission-attribution`（admin-only）
- `POST /api/vehicles` 新增必填 `purchase_agent_id`
- `POST /api/vehicles/{vehicle}/reserve`：admin／manager 必填 `sales_agent_id`；sales 不接受自行指定，由後端保存本人

完整契約已同步至 `backend/API.md`。

## 對立性審查修正

- 已修正 fresh seed／v1.2 升級後因沒有 Salary Profile 而無法建車的 bootstrap 死結；不新增推測式薪資回填。
- 已修正 `commission_enabled=false` 或尚無 Salary Profile 的 sales 無法保留車輛；事實歸屬與獎金資格已解耦。
- `commission-agent-options` 已收緊為 admin／manager，且清單只依 active user 產生，不洩漏獎金啟用狀態。
