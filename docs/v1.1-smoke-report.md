## 2. 驗收前最後三個修正 commit

```text
7bb5cc2 fix: 收斂老闆核准流程與 sales 銷售收款可視範圍
a42eaee fix: 補齊成交結案待審收支檢查與收支單筆查詢授權漏洞
e05cf8d fix: 核准訂金/尾款/退款前檢查車輛是否已結案，防止事後核准打破已關帳不變量
```

### 2.1 `7bb5cc2`

主要內容：

- sales 可看開價 / 底價 / 成交價
- sales 車輛詳情新增銷售收款摘要
- sales 可看自己上報的整備支出與審核狀態
- sales 不看收購價、成本、毛利、資金帳戶
- manager 可看完整營運與毛利，但不可核准 / 駁回
- admin 才能核准 / 駁回
- `manual` / `vehicle_shortcut` / `vehicle_workflow` 若建立者不是 admin，皆進 pending

### 2.2 `a42eaee`

主要內容：

- 成交結案前若仍有 pending 訂金 / 尾款 / 退款，會阻擋結案
- 收支單筆查詢補上授權邊界，避免 sales 透過 detail endpoint 讀到不該看的收支

### 2.3 `e05cf8d`

主要內容：

- 已售出 / 已取消車輛，不可再事後核准 pending 訂金 / 尾款 / 退款
- 防止「先成交結案，後核准退款」導致已關帳車輛收款不足
- 保護已關帳金額不變量

## 3. 人工 smoke 範圍

### 3.1 Admin 主流程

通過項目：

1. Dashboard 顯示
2. 建立賣方客戶
3. 建立買方客戶
4. 建立車輛
5. 建車同步購車付款

7. 車輛列表與詳情
8. 上架

10. 收尾款
11. 老闆核准收款
12. 成交結案
13. 列印 / 收支資料檢查
14. 稽核紀錄檢查

### 3.2 Sales 權限 smoke

使用者確認無問題的重點：

- sales Dashboard 不再只剩車輛列表
- sales 可看車輛列表
- sales 可看客戶列表 / 新增客戶
- sales 可進收支紀錄與新增一般收入 / 支出
- sales 可看開價
- sales 可看底價
- sales 可看成交價
- sales 可看銷售收款摘要
- sales 可看自己上報支出與審核狀態
- sales 不可看收購價
- sales 不可看完整成本
- sales 不可看毛利
- sales 不可看資金帳戶餘額
- sales 不可看稽核紀錄

### 3.3 Manager 權限 smoke

通過重點：

- manager 可看完整營運金額
- manager 可看完整車輛成本與毛利
- manager 可管理車輛流程
- manager 不可核准 / 駁回收支
- manager 不可管理使用者
- manager 不可查看稽核紀錄
- manager 不可管理資金帳戶

### 3.4 Admin 核准流程 smoke

通過重點：

- sales / manager 建立會影響正式金額的收支後進 pending
- admin 核准後才計入正式餘額、成本、毛利
- admin 駁回後不計入正式彙總
- approved / rejected 不可逆
- legacy_unknown 不可核准 / 駁回

### 3.5 成交結案 smoke

通過重點：

- sales 收訂金 → pending
- sales 收尾款 → pending
- admin 尚未核准前，成交結案會被擋
- admin 核准足額訂金 / 尾款後，成交結案成功
- 若有 pending 退款，成交結案會被擋
- 已結案車輛不可再事後核准訂金 / 尾款 / 退款

## 6. 已知限制與不納入 v1.1 的項目

v1.1 不處理：

- 正式會計
- 會計科目 / 傳票
- 稅務申報
- 發票
- POS
- OCR
- 記帳士匯出
- 複雜角色權限勾選 UI
- SaaS 多租戶
- 手機版深度優化

已知可留到 v1.1.1 / v1.2 的改善：

- 全站表單 per-field error / on-blur validation 尚未統一
- 列印頁建議正式使用前再肉眼檢查紙本排版
- 待審核收支若實務量變大，應做專門的「待審核工作台」
- sales 手機操作可能需要再優化按鈕與表單順序
- 憑證附件 / 收據照片尚未納入正式 v1.1 封版

## 7. 建議封版指令

建議在確認本文件 commit 後打 tag：

```bash
git tag v1.1-smoke-passed
git push origin v1.1-smoke-passed
```

封版後的建議作法：

1. 先用真實車行流程試跑 3–7 天
2. 只記錄實務摩擦點，不立即開大功能
3. 以 v1.1.1 小修方式處理文案、按鈕、欄位、預設值、空狀態、手機畫面
4. 試跑穩定後再規劃 v1.2

## 8. 下輪 v1.1.1 建議檢查清單

真實試跑時建議記錄：

- 哪個按鈕不好找
- 哪個欄位名稱看不懂
- 哪個表單預設值不順
- sales 用手機上報支出是否麻煩
- 老闆核准收支是否需要更快入口
- 車輛詳情頁資訊是否過多或不足
- Dashboard 是否能一眼看出今天該處理什麼
- 列印資料是否符合紙本資料夾需求

不要把這些直接升成 v1.2 大功能；先收斂成 v1.1.1 小修清單。
