## 3. API、URL 與權限基線

### Dashboard

- `GET /api/dashboard/summary` 由薄 Controller 呼叫 `DashboardService`，Service 以 transaction／REPEATABLE READ 提供同一資料快照。
- 現況只提供帳戶餘額、本月收支、淨流入、各車況與本月成交；v1.4 工作 KPI、毛利與 30 天趨勢尚未實作。
- `sales` 現況由 Controller 移除財務欄位；unknown role 同樣走非財務輸出。第 3 部分仍需依 v1.4 白名單 DTO／Resource 契約收斂。

### Vehicles

- 前端 `VehicleList` 目前只有 component state，沒有 URL 還原；API type 只接受單一 `status`。
- `IndexVehicleRequest` 只接受單一合法狀態；`VehicleService::listVehicles()` 只套單一狀態，尚未支援多狀態或 `is_preparation_completed`。
- 列表目前 eager load 收／賣車人，不載入封面；`VehicleResource` 回傳完整內部車輛欄位並依角色遮蔽價格。
- `Vehicle::photos()` 只包含已提交可見照片，且依 `sort_order` 排序；第 5 部分可在不改 schema 下加封面關聯／受限 eager load。
- `sales` 目前可看 `asking_price`、`floor_price`、`sold_price`。v1.4 Vehicle Card 規格要求 sales 卡片不呈現底價，這是列表呈現差異；後端既有詳細頁議價契約不可因此回歸。

### Money Entries

- 後端已支援搜尋、日期、方向、分類、帳戶、車輛、`approval_status` 與分頁。
- 前端目前全部使用 component state，且 `MoneyEntryList` 為取得車輛選項直接呼叫 `apiClient` 與 `per_page=100`；第 2 部分需將 URL 還原與 API 集中化一併處理。
- `salary_settlement`、sales 可見範圍與 cash account 遮蔽已由 Service／Policy／Resource 保護，v1.4 不可改變。

### Customers、Layout 與流程重用

- 建立車輛、建立客戶、一般收支建立頁與車輛詳情整備支出流程都已存在；Dashboard 快捷操作只能導向這些既有頁面。
- `AppLayout` 現有 Header 只包含 Mobile Sidebar 開關、使用者、主題與登出；v1.4 不在 Header 加快捷操作或通知。
- 本階段未新增 route、API、migration、schema、Workflow、狀態、角色或權限。
