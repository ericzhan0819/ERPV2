## 本次完成

- `AppLayout` 在小於 `lg` 的尺寸改用 overlay drawer，桌機維持 224px 固定 Sidebar；手機可由 Header 開啟、遮罩或關閉按鈕收合，換頁後自動收合。
- 主內容補 `min-w-0`，手機 padding 降為 16px；導覽、按鈕、一般輸入框與 select 的觸控高度至少 44px。
- 薪資月份與待補歸屬在手機使用卡片版面；薪資詳情的項目明細、薪資設定、獎金級距、發薪區及加扣項 modal 改為可堆疊版面。
- 本次表單控制項以 `.form-control-touch` 明確 opt-in 既有 `surface`、`foreground`、`border` token 與 44px 高度，並以 `color-scheme` 支援原生 select／option 的 light／dark 呈現與 disabled 狀態；未用裸元素選擇器改變其他模組的 compact controls。
- `eligible_sales_count` 改標示為「獎金級距計入台數」；未啟用獎金時不再顯示升級提示；`money_entry_source_review` 改顯示系統管理處理說明，不再連到無法修正來源的車輛頁。
- 待補歸屬頁新增獨立 loading state，API 尚未完成時不顯示 empty state。
- 草稿員工卡的獎金啟用狀態改讀既有 admin-only Salary Profiles API；`commission_warnings` 只作載入前 fallback，停用且 0 台時也不會誤顯示升級提示。

## Review 修正

收到對立性審查後逐項查證並確認兩項 finding 有效：

1. 原先裸 `button`／`input`／`select`／`textarea` 規則會讓 CustomerSelect 行內操作、收支核准／駁回及分頁等 compact controls 被 `min-height: 44px` 撐高。現已移除裸元素規則，改為 `.form-control-touch` opt-in。瀏覽器以原受影響 class 驗證後，高度為 16px／26px／38px，`min-height` 均為 0。
2. `commission_warnings` 在停用獎金且 0 台時沒有資料，不能作為權威狀態。草稿詳情現會讀取 Salary Profiles API 的 `commission_enabled`；載入中或失敗時不會先顯示升級提示，confirmed／paid 仍只顯示已鎖定語意。

修正後重新執行 5 個 viewport × 2 個 theme × 5 個頁面，共 50 組瀏覽器檢查，結果為 0 failure。
