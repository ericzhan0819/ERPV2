## 7. Smoke 過程發現並已修復的問題

### 7.1 MySQL database cache lock refresh 誤判

症狀：

```text
車輛照片上傳在真實 MySQL 環境下 100% 回報「照片處理逾時，請重新上傳」
```

根因：

Laravel database cache lock 的 `refresh()` 依 PDO affected rows 判斷是否成功。MySQL 預設只回報實際值有變動的列；同一秒內續約時 expiration 值可能相同，因此回報 0，導致程式誤判鎖已遺失。

修復：

在 MySQL／MariaDB PDO options 加入：

```php
Mysql::ATTR_FOUND_ROWS => true
```

修復後真實 MySQL 上傳、封面、排序、刪除流程均通過。

Commit：

```text
fdb5da5 fix: 修復 MySQL 環境下車輛照片上傳因 database cache lock refresh 誤判逾時而 100% 失敗的問題
```

### 7.2 Public storage symbolic link 缺失

症狀：

```text
/storage/vehicles/...webp 回傳 403
```

處理：

```bash
cd ~/ERPV2/backend
php artisan storage:link
```

完成後主圖與縮圖均回傳 `200 image/webp`。

此項屬部署／環境初始化步驟，README 已記錄，不是程式碼缺陷。
