## 3. 靜態稽核結論

- 業務 API endpoint 仍集中在 `frontend/src/api`；頁面與元件只 import API modules。
- v1.4 新增硬編碼色彩只出現在集中式 Design Token 與同步瀏覽器 `theme-color` 的既定 light／dark 值；既有列印專用 CSS 與未使用的 Vite asset 不屬 v1.4 工作區元件樣式。
- Dashboard 沒有 Table、車輛列表或展開明細；只提供 KPI／趨勢與 URL 導流。
- 前端只格式化後端正式金額、映射後端趨勢點並計算 SVG 座標，沒有自行重算收入、支出、餘額或毛利。
- `DashboardSummaryResource` 使用角色白名單回傳財務欄位；sales／未知角色原始 JSON 缺欄測試與 Vehicle list 敏感欄位測試均在回歸中通過。
- `v1.3-smoke-passed..HEAD` 沒有 `backend/database/migrations` 或 schema 檔案差異。

## 4. 未執行與原因

- 15 個完整 suite skip 為既有 environment-gated 測試；本階段要求的真實 MariaDB 時區邊界已另行啟用並通過。v1.4 沒有 transaction、idempotency、薪資或 Customer concurrency 邏輯變更，因此未重跑其餘無關的破壞性 MariaDB 並發套件。
- 未執行第 11 部分三角色 browser manual smoke；需要瀏覽器與使用者操作驗收，不能以自動測試冒充通過。
- 未執行手機實機注音／倉頡 composition 與 iOS Safe Area；需要真實輸入法及 iOS 裝置，PLAN 對應項目維持未勾選。
