# v1.3 第 5 部分交接：整月跨級獎金計算器

日期：2026-07-13

## 完成範圍

`PLAN_v1.3.md` 第 5 部分已完成：

- 新增無寫入副作用的 `SalaryCommissionCalculator`，集中處理薪資獎金公式。
- 批次只讀取傳入車輛的 approved MoneyEntry；pending／rejected 不計入。
- 單車計算回傳收入、支出、毛利、公司營運保留、分配池、收車獎金、賣車獎金、公司剩餘與公司最終取得。
- 毛利小於或等於 0 時保留原毛利供明細顯示，但所有自動獎金皆為 0，不產生負獎金。
- 同一批 eligible 車輛依 `sales_agent_id` 分組，從方案 tiers 套用整月跨級比例；第 5 台成立後，同一人的前 4 台會一併使用 50%。
- 回傳每位賣車人的符合資格台數、目前級距、距下一級台數、目前賣車獎金合計與既有車輛跨級後預估增加金額。
- basis points 計算全部向下取整，餘數由公司剩餘分配額吸收，正毛利分配總和恆等於毛利。
- 比例乘法使用商數／餘數拆算，避免大額 `amount × bps` 中間值 overflow；彙總若超過 PHP 安全整數範圍會拋出明確例外。

## 計算器邊界

第 5 部分只負責公式，不負責決定車輛資格。呼叫端必須傳入同一結算月份、已通過資格檢查的車輛集合；計算器不會靜默略過傳入資料，也不會推定收車人／賣車人。第 6 部分需新增集中資格服務，檢查 sold 狀態、sold_at 月份、收付款完整性、pending／legacy_unknown、歸屬與跨月份重複引用，再把合格集合交給計算器。

同一人兼任收車人與賣車人時，逐車結果保留兩個 agent id 與兩種獎金；後續建立 settlement items 時可各自歸入該員工。不同人時則依各欄位分開歸屬。
