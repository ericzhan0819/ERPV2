# Claude Code 實作任務：中古車行內部營運系統 1.0

請依照本文件實作一套前後端分離的中古車行內部營運系統 1.0。

本任務只實作本文件明確列出的功能。不得自行延伸、不得加入額外模組、不得實作未列出的進階功能。

核心要求：

1. 前後端分離。
2. 功能必須模組積木化，方便未來維護、升級、擴展。
3. 後端負責所有正式業務邏輯與金流計算。
4. 前端只負責操作介面、API 呼叫與資料呈現。
5. 不做正式會計、不做報稅、不做發票、不做 POS。
6. 不使用假資料假裝功能完成。

---

## 1. 系統定位

本系統是小型中古車行內部營運管理系統。

核心流程：

```
車輛買入
→ 建立車輛資料
→ 狀態進入整備中
→ 記錄購車付款與整備支出
→ 整備完成並上架
→ 收訂金並保留
→ 收尾款
→ 成交結案
→ 列印該車收支明細
```

系統需要即時顯示：

```
現金目前多少
銀行目前多少
其他帳戶目前多少
每台車收了多少
每台車花了多少
每台車毛利多少
本月收入多少
本月支出多少
本月成交幾台車
```


## 9. 金流計算規則

## 9.1 帳戶餘額

```
目前餘額 = 期初餘額 + 收入總額 - 支出總額
```

收入總額：

```
money_entries.direction = income
且 cash_account_id = 該帳戶
```

支出總額：

```
money_entries.direction = expense
且 cash_account_id = 該帳戶
```

禁止手動儲存目前餘額。

---

## 9.2 單車收入

```
單車收入 = 該車所有 income money_entries 合計
```

包含：

```
訂金收入
尾款收入
其他單車收入
```

---

## 9.3 單車支出

```
單車支出 = 該車所有 expense money_entries 合計
```

包含：

```
購車付款
維修支出
美容支出
代辦支出
拍場支出
退款
其他支出
```

---

## 9.4 單車毛利

```
單車毛利 = 單車收入合計 - 單車支出合計
```

---

## 9.5 本月統計

本月收入：

```
entry_date 在本月
direction = income
amount 合計
```

本月支出：

```
entry_date 在本月
direction = expense
amount 合計
```

本月淨流入：

```
本月收入 - 本月支出
```

本月成交台數：

```
vehicles.status = sold
sold_at 在本月
```

## 12. 車輛流程

## 12.1 新增買入車輛

```
新增車輛
→ 系統建立 vehicles
→ 系統自動產生 stock_no
→ status = preparing
→ 導向車輛詳情頁
```

---

## 12.2 新增購車付款

```
在車輛詳情點「新增購車付款」
→ 輸入日期、金額、付款帳戶、付款對象、備註
→ 系統建立 money_entries
→ direction = expense
→ category = 購車付款
→ vehicle_id = 該車
→ 帳戶餘額重新計算
→ 單車支出增加
```

---

## 12.3 新增單車支出

```
在車輛詳情點「新增支出」
→ 選擇維修 / 美容 / 代辦 / 拍場 / 其他
→ 輸入日期、金額、付款帳戶、對象、備註
→ 系統建立 money_entries
→ direction = expense
→ vehicle_id = 該車
→ 帳戶餘額重新計算
→ 單車支出增加
```

---

## 12.4 整備完成並上架

```
在整備中車輛點「整備完成並上架」
→ 輸入開價、底價、上架日期、銷售備註
→ 系統更新 vehicles
→ status = listed
```

---

## 12.5 收訂金並保留

```
在上架中車輛點「收訂金並保留」
→ 輸入買方姓名、電話、成交價、訂金金額、收款帳戶、備註
→ 系統建立 money_entries
→ direction = income
→ category = 訂金收入
→ vehicle_id = 該車
→ 系統更新 vehicles
→ status = reserved
```

---

## 12.6 收尾款

```
在保留中車輛點「收尾款」
→ 輸入尾款金額、收款帳戶、備註
→ 系統建立 money_entries
→ direction = income
→ category = 尾款收入
→ vehicle_id = 該車
→ 單車收入增加
```

若訂金 + 尾款不等於成交價：

```
顯示提醒
不阻擋送出
```

---

## 12.7 成交結案

```
在保留中車輛點「成交結案」
→ 系統檢查已有成交價與買方資料
→ 系統檢查至少已有一筆訂金或尾款收入
→ 系統更新 vehicles
→ status = sold
→ sold_at = 使用者輸入日期或今天
→ 可列印成交結案收支明細
```
